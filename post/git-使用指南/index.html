简介

Git是分布式版本控制软件

命令

git init

git init [directory]

释义

创建一个空 Git 仓库或者重新初始化一个已存在的仓库

示例

git init my-project

git clone

git clone [--verbose | -v] [--progress] [--origin | -o <origin>] [--branch | -b <branch>] [--depth <depth>] <repository> [<directory>]

释义

创建一个空 Git 仓库或者重新初始化一个已存在的仓库

参数

参数释义
verbose显示详情
progress显示进度
origin使用指定的名称追踪远程仓库，而不是使用默认的名称origin
branch指定要克隆的分支
depth指定commit的历史记录数量

示例

git clone -v --progress --origin origin --branch master --depth 1 git@github.com:jason207010/git-test.git

git config

释义

设置配置参数，默认是当前仓库的配置参数

参数

–global 编辑全局配置参数 –system 编辑系统参数

常用配置参数名

参数名释义
user.name用户名
user.email邮箱

示例

git config user.name "jason"
git config --global user.email "jason@mail.com"

git config -l

释义

列出所有配置参数

git config -e

释义

打开一个 vi 编辑器来编辑配置参数，默认打开的是当前仓库的配置

参数

–global 编辑全局配置参数 –system 编辑系统参数

git add

git add [--verbose | -v] [--force | -f] [<pathspec>...]

释义

把工作区指定文件或目录添加到暂存区

参数

参数释义
verbose显示详情
force强制添加，把.gitignore文件中定义的忽略文件也添加到暂存区

示例

git add .

git rm

git rm [-f | --force] [-r] [--cached] <file>...

释义

从工作区，或工作区和暂存区删除文件

参数

参数释义
force强制删除，覆盖 up-to-date 检查
r递归删除，在删除文件夹的时候使用递归删除
cached从暂存区删除文件，但保留工作区的

示例

git rm --cache test.txt

git mv

git mv [-v | --verbose] [-f | --force] <source> <destination>

释义

移动或重命名文件、目录或链接

参数

参数释义
verbose显示详情
force强制移动或重命名，如果目标文件已存在，则替换这个文件

示例

git mv test.txt test1.txt

git commit

git commit [-m <msg> | --message=<message>] [-v | --verbose] [<file>...]

释义

把暂存区有改动的文件提交到仓库中

参数

参数释义
message本次提交的信息
verbose显示详情

示例

git commit -m 'test message'

git status

释义

显示工作区和暂存区的状态

git show

释义

显示 tag/commit/branch 的信息

示例

# 显示分支信息
git show master
# 显示标签信息
git show mytag
# 显示 commit 信息
git show 4743b5660b3853b2641f654268049977f09f2da7

git pull

释义

把远程仓库下载到本地并合并，git pull = git fetch + git merge

git fetch

释义

把远程仓库下载到远程仓库本地副本，后续可通过 git merge 合并到本地仓库

git push

git push [--all | --mirror | --tags] [-v | --verbose] [-d | --delete] [-u | --set-upstream] [<repository> [<refspec>...]]

释义

把本地仓库推送到远程

参数

参数释义
all推送所有分支到远程，不包括分支
tags推送所有标签到远程
verbose显示详情
delete删除远程分支/标签
set-upstream设置远程仓库与本地关联

示例

# 推送 tag 到远程
git push --tags
# 在远程仓库创建分支，并把当前分支推送到远程这个分支上
git push --set-upstream origin test
# 推送分支到远程
git push origin test
# 推送指定 tag 到远程
git push origin mytag
# 删除远程分支
git push --delete origin test
# 删除远程标签
git push --delete origin mytag

git branch

git branch [-r | --remotes] [-a | --all] [--list] [-v | -vv | --verbose] [<pattern>...]

git branch [--set-upstream-to=<remote/branchname> | -u <remote/branchname>] [<branchname>]

git branch --unset-upstream [<branchname>]

git branch [-m | --move] [-f | --force] [<oldbranch>] <newbranch>

git branch [-d | --delete] [-f | --force] [-r | --remotes] <branchname>...

释义

分支操作

参数

参数释义
list列出分支，可以与正则表达式配合使用，如：git branch –list ‘maint-*’
all列出所有分支，包括远程分支和本地分支
verbose显示详情
set-upstream-to关联远程分支与本地分支
unset-upstream取消远程分支与本地分支关联
move移动/重命名分支
delete删除分支
force强制重命名/删除分支
remotes远程分支操作，列出/删除远程分支

示例

# 新建分支
git branch test
# 把本地分支推送到远程仓库
git push --set-upstream origin test
# 删除本地分支
git branch --delete test
# 删除远程分支
git branch --delete --remotes origin/test

git checkout

git checkout [-f | --force] [-m | --merge] [-b] [<branch>] [<start_point>]

git checkout [<tree-ish>] [<pathspec>...]

释义

  1. 切换分支

2.从暂存区/commit/tag/HEAD 恢复工作区文件

示例

# 从 暂存区 或 HEAD 重置文件
git checkout test.txt
# 从 HEAD 重置文件
git checkout HEAD test.txt
# 从 tag 重置文件
git checkout mytag test.txt
# 从 commit 重置文件
git checkout 4743b5660b3853b2641f654268049977f09f2da7 test.txt

参数

参数释义
force当有未 commit 的文件时，强制切换分支，这会导致未 commit 的文件内容丢失
merge切换分支的时候，把当前分支合并到目标分支
b新建一个分支，同时切换到该分支

示例

# 切换到指定分支
git checkout test
# 基于指定 commit 新建分支，并切换到该分支
git checkout -b test 4743b5660b3853b2641f654268049977f09f2da7
# 基于指定 tag 新建分支，并切换到该分支
git checkout -b test mytag

git merge

git merge [<commit>...]
git merge [<branch>]

释义

合并指定的 commit 或分支到当前分支

示例

# 合并指定 commit 到当前分支
git merge f50f0f3939cb6054b7b5ee5d1a13f398d478296c
# 合并指定分支到当前分支
git merge master

git cherry-pick

git cherry-pick <commit>...

释义

合并指定的 commit 到当前分支的工作区，与 git merge 不同的是，git cherry-pick 会把指定 commit 到当前分支工作区，需要手动 git add，git commit

示例

git cherry-pick 01fa2af3e1ca6235bad32ca6ef8303c75a2a347a

git tag

git tag [-l | --list] [<pattern>...]

git tag [-f | --force] <tagname> [<commit> | <object>]

git tag -d | --delete <tagname>...

释义

标签操作

参数

参数释义
list列出所有 tag
force强制创建 tag，这会覆盖原有的同名 tag
delete删除 tag

示例

# 在指定 commit 创建 tag
git tag mytag 4743b5660b3853b2641f654268049977f09f2da7
# 列出所有 tag
git tag
# 查看 tag 信息
git show mytag

git log

git log [--stat] [-S<string>] [-G<regex>] [--grep=<pattern>] [--merges] [--no-merges] [<path>...]

释义

日志

参数

参数释义
stat显示文件变化统计
S搜索指定关键字
G匹配指定的正则表达式
grep匹配指定模式
merges显示合并相关的日志
no-merges不显示合并相关的日志

git diff

git diff [<commit>] [<path>...]

git diff --cached [<commit>] [<path>...]

释义

文件变化比较

示例

# 显示工作区和暂存区的文件变化
git diff
# 显示暂存区和上一个 commit 的文件变化
git diff --cached
# 显示指定文件工作区和指定 commit 的文件变化
git diff 01fa2af3e1ca6235bad32ca6ef8303c75a2a347a test.txt
# 显示指定分支的文件变化
git diff master test

git remote

git remote [-v | --verbose]

git remote add <name> <url>

git remote rename <old> <new>

git remote remove <name>

git remote [-v | --verbose] show <name>...

释义

远程仓库管理

参数

参数释义
verbose显示详情

示例

# 添加远程仓库
git remote add origin git@github.com:jason207010/git-test.git
# 重命名远程仓库
git remote rename origin myorigin
# 显示远程仓库详情
git remote -v show myorigin
# 删除远程仓库
git remote remove myorigin

git reset

git reset [<tree-ish>] <paths>...

git reset [--soft | --mixed [-N] | --hard | --merge | --keep] [<com
mit>]

释义

重置操作

参数

参数释义
soft重置本地仓库 HEAD 指针到指定 commit，但工作区、暂存区保持不变
hard重置本地仓库 HEAD 指针、工作区、暂存区到指定 commit
mixedreset 命令的默认参数，重置本地仓库 HEAD 指针、暂存区到指定 commit，但工作区保持不变
keep如果工作区有未 add 到暂存区的文件，重置本地仓库 HEAD指针、暂存区到指定commit，但工作区保持不变
否则重置本地仓库 HEAD 指针、暂存区、工作区到指定 commit

示例

# 如果工作区有修改，重置暂存区指定文件与仓库 HEAD 一致，工作区保持不变
# 如果工作区没有修改，重置暂存区、工作区与仓库 HEAD 一致
git reset test.txt
# 如果工作区有修改，重置暂存区指定文件与指定 tag 一致，工作区保持不变
# 如果工作区没有修改，重置暂存区、工作区与指定 tag 一致
git reset mytag test.txt
# 如果工作区有修改，重置暂存区指定文件与指定 commit 一致，工作区保持不变
# 如果工作区没有修改，重置暂存区、工作区与指定 commit 一致
git reset fdac8342a1ac278b17ec13772e10056a41c4e745 test.txt

git stash

git stash list

git stash show [<stash>]

git stash drop [<stash>]

git stash ( pop | apply ) [<stash>]

git stash clear

释义

把工作区、暂存区的变化暂时储存起来，使工作区、暂存区与 HEAD 指针保持一致

示例

# 把工作区、暂存区的变化储存起来
git stash
# 显示所有 stash
git stash list
# 把 stash 的文件变化应用到工作区
git stash pop
# 清空所有 stash
git stash clear

Windows Git 设置代理

echo 'ProxyCommand "C:\Program Files\Git\mingw64\bin\connect.exe" -S 127.0.0.1:10808 %h %p'>~/.ssh/config

Mac Git 设置代理

vi ~/.ssh/config

# 必须是 github.com
Host github.com
   HostName github.com
   User git
   # 走 HTTP 代理
   # ProxyCommand socat - PROXY:127.0.0.1:%h:%p,proxyport=8080
   # 走 socks5 代理（如 Shadowsocks）
   ProxyCommand nc -v -x 127.0.0.1:1080 %h %p
