Java

Servlet、Filter 的 url-pattern 匹配规则

精确匹配

@WebServlet(urlPatterns = "/test")
public class TestServlet extends HttpServlet {
}

扩展名匹配

@WebServlet(urlPatterns = "*.do")
public class TestServlet extends HttpServlet {
}

路径匹配

@WebServlet(urlPatterns = "/test/*")
public class TestServlet extends HttpServlet {
}

任意url匹配

@WebServlet(urlPatterns = "/*")
public class TestServlet extends HttpServlet {
}

优先顺序

当一个url与多个servlet的匹配规则可以匹配时，则按照 “ 精确路径 > 最长路径>扩展名”这样的优先级匹配到对应的servlet

注意事项

  • “路径匹配”和“扩展名匹配”无法同时设置，否则tomcat启动报错
  • @WebServlet(urlPatterns = “/test/*/test”) 是精确匹配，这里的*不是通配的含义

Spring Boot 注册 Servlet 和 Filter

通过配置 ServletRegistrationBean 注册

  • 编写 Servlet

    public class TestServlet extends HttpServlet {
}
    • 配置 ServletRegistrationBean
    @SpringBootApplication
public class SpringBootDemoApplication {

    public static void main(String[] args) {
    SpringApplication.run(SpringBootDemoApplication.class, args);
    }

    @Bean
    public ServletRegistrationBean<TestServlet> testServletServletRegistrationBean() {
    ServletRegistrationBean<TestServlet> bean = new ServletRegistrationBean<>();
    bean.addUrlMappings("/test");
    bean.setServlet(new TestServlet());
    return bean;
    }
}

注解方式注册

  • 编写 Servlet

    Servlet 使用 @WebServlet 注解

    @WebServlet(urlPatterns = "/test")
public class TestServlet extends HttpServlet {
}
    • 配置 Spring Boot 扫描 Servlet

    @ServletComponentScan 注解会扫描 有 @WebServlet 注解的 Servlet，并注册进来

    @SpringBootApplication
@ServletComponentScan
public class SpringBootDemoApplication {

    public static void main(String[] args) {
    SpringApplication.run(SpringBootDemoApplication.class, args);
    }
}

读取项目 resources 目录下的文件

//根目录就是 resources 目录
this.getClass().getResource("/application.properties");
//相对路径根目录就是 resources 目录
this.getClass().getClassLoader().getResource("application.properties");

Spring MVC 中 Jackson 反序列化时区问题

方法一

在注解中指定时区，但每一个字段都要指定，操作繁琐，而且容易遗漏，不推荐

@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss", timezone = "GMT+8")
private Date startTime;

方法二

Spring Boot 框架会自动帮我们配置相关的 Bean，通过查看源码发现，Spring Boot 在 JacksonAutoConfiguration 这个类中自动配置了 Jackson 相关 Bean，我们只需要手动设置 Bean 即可

@Bean
public Jackson2ObjectMapperBuilderCustomizer jackson2ObjectMapperBuilderCustomizer() {
    return jacksonObjectMapperBuilder -> jacksonObjectMapperBuilder.timeZone("GMT+8");
}

方法三

继续细看方法二中的相关源码，可以发现时区相关的设置是从 Spring 配置的 spring.jackson.time-zone 字段读取，我们只需要配置一下即可，编辑 application.properties 文件

spring.jackson.time-zone=GMT+8

Maven

Maven Dependency Scope

Dependency scope 是用来限制Dependency的作用范围的, 影响maven项目在各个生命周期时导入的package的状态

compile

默认的scope，表示 dependency 都可以在生命周期中使用。而且，这些dependencies 会传递到依赖的项目中

provided

跟compile相似，但是表明了dependency 由JDK或者容器提供，例如Servlet AP和一些Java EE APIs。这个scope 只能作用在编译和测试时，同时没有传递性。使用这个时，不会将包打入本项目中，只是依赖过来

runtime

表示dependency不作用在编译时，但会作用在运行和测试时

test

表示dependency作用在测试时，不作用在运行时

system

跟provided 相似，但是在系统中要以外部JAR包的形式提供，maven不会在repository查找它

<dependency>
　<groupId>javax.runtime</groupId>
　<artifactId>rt</artifactId>
　<version>1.8</version>
　<scope>system</scope>
　<systemPath>${java.home}/lib/rt.jar</systemPath>
</dependency>

import

它只使用在中，表示从其它的pom中导入dependency的配置

导出依赖 jar 包

mvn dependency:copy-dependencies
参数 释义
outputDirectory 指定 jar 包导出目录
includeScope 指定依赖级别

示例：

mvn dependency:copy-dependencies -DoutputDirectory=lib   -DincludeScope=compile

查看依赖树

mvn dependency:tree

NPM

设置淘宝 NPM 镜像

临时使用

npm --registry https://registry.npm.taobao.org install express

长久使用

npm config set registry https://registry.npm.taobao.org

使用 cnpm

npm install -g cnpm --registry=https://registry.npm.taobao.org
cnpm install express

恢复默认镜像

npm config set registry https://registry.npmjs.org

MySQL

sql_safe_updates 参数

sql_safe_updates 开启状态下会限制 update、delete 操作中不带 where 条件的SQL 执行。在没有 where 条件或者 where 后不是索引字段时，必须使用 limit ；在有 where 条件时，where 为索引字段，以此确保不会发生全表更新的问题

查看 sql_safe_updates 状态

show variables like 'sql_safe_updates';

关闭 sql_safe_updates

set sql_safe_updates=0;
