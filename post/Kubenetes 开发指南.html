安装 Kubeadm

Ubuntu

1
2
3
4
5
6
7

  apt-get update && apt-get install -y apt-transport-https
  curl https://mirrors.aliyun.com/kubernetes/apt/doc/apt-key.gpg | apt-key add - 
  cat <<EOF >/etc/apt/sources.list.d/kubernetes.list
  deb https://mirrors.aliyun.com/kubernetes/apt/ kubernetes-xenial main
  EOF  
  apt-get update
  apt-get install -y kubelet kubeadm kubectl

  repo_gpgcheck=1

  EOF
  setenforce 0
  yum install -y kubelet kubeadm kubectl
  systemctl enable kubelet && systemctl start kubelet

初始化 master 节点

在 master 服务器执行命令：

1

  kubeadm init

也可以先导出默认初始化配置，再根据实际需要修改配置，然后执行 kubeadm init：

1

  kubeadm config print init-defaults > kubeadm-init.yaml

1

  kubeadm init --config kubeadm-init.yaml

命令执行完会显示一行信息：

1
2

  kubeadm join 192.168.43.207:6443 --token jgdah6.2ckbsm793nqvz30f \
      --discovery-token-ca-cert-hash sha256:17795b4294005f36c07a3fa50a1161fea4fed67e83205ef2d5f5d1c47458e41d

修改环境变量

如果不修改环境变量，使用 kubectl 命令时会报以下异常：

1

  The connection to the server localhost:8080 was refused - did you specify the right host or port?

master 节点

1
2
3

  echo "export KUBECONFIG=/etc/kubernetes/admin.conf" >> ~/.bashrc

  source ~/.bashrc

node 节点

从 master 节点复制配置文件到 node 节点

1

  scp root@192.168.43.207:/etc/kubernetes/admin.conf /etc/kubernetes/

1
2
3

  echo "export KUBECONFIG=/etc/kubernetes/admin.conf" >> ~/.bashrc

  source ~/.bashrc

部署 flannel 网络插件

此时 master 节点和 node 节点还无法正常连接，master 节点和 node 节点都是 NotReady 状态，需要安装 flannel 网络插件使两个节点能够正常连接

在 master 节点执行命令：

1

  kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml
