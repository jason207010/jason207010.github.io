Dockerfile

Dockerfile 指令必须大写

FROM

FROM 指令初始化一个新的构建阶段，同时为后续指令指定一个基本镜像，因此，一个合法的 Dockerfile 必须以 FROM 指令开始

用法

1

  FROM <image> [AS <name>]

或

1

  FROM <image>[:<tag>] [AS <name>]

或

1

  FROM <image>[@<digest>] [AS <name>]

参数

参数名 释义
image 镜像名
tag 镜像标签
digest 镜像数字签名，使用 docker images ––digests 查看

示例

1
2

  ARG CENTOS_VERSION=latest
  FROM centos:${CENTOS_VERSION}

RUN

RUN 指令用来指定容器构建时要运行的命令，这些命令是在容器内执行的

用法

1

  RUN <command>

或

1

  RUN ["executable", "param1", "param2"]

示例

1

  RUN ["/bin/bash","-c","echo runing > log.txt"]

1

  RUN echo runing >> log-1.txt

CMD

CMD 指令用来指定容器启动时的命令，这和 docker run 命令非常类似，如果 docker run 指定了执行命令，将会覆盖 CMD 指令的命令。 这些命令是在容器内执行的，一个 Dockerfile 只能有一个 CMD 指令，如果有多个，则只执行最后一个

1

  CMD /bin/bash

等价于

1

  docker run -t -i centos /bin/bash

用法

1

  CMD ["executable","param1","param2"]

或

1

  CMD ["param1","param2"]

或

1

  CMD command param1 param2

示例

1

  CMD /bin/bash

LABEL

LABEL 指令为镜像添加元数据，可通过 docker inspect 命令查看

用法

1

  LABEL <key>=<value> <key>=<value> <key>=<value> ...

示例

1

  LABEL label1=label1 label2="this is lable2"

EXPOSE

EXPOSE 指令用于声明容器将暴露的端口及协议类型，实际上并未发布端口，它充当构建镜像的人员与运行容器的人员之间的一种文档类型，运行容器时使用 docker run -p 来映射容器端口和宿主端口的绑定 可以使用 docker inspect 命令查看镜像暴露的端口

用法

1

  EXPOSE <port> [<port>/<protocol>...]

参数

参数名 释义
port 端口
protocol 通信协议，TCP 或 UDT，默认TCP

示例

1

  EXPOSE 8080

ENV

ENV 指令用于设置环境变量，可以使用 docker inspect 命令查看镜像设置的环境变量

1

  ENV key1=value1 key2=value2

等价于

1

  docker run --env key1=value1 --env key2=value2

用法

1

  ENV <key> <value>

或

1

  ENV <key>=<value> ...

示例

1

  ENV hello world

ADD

ADD 指令复制宿主上的文件，或远程文件到容器中，如果源文件是压缩文件（identity, gzip, bzip2 或 xz 格式），ADD 指令会将源文件解压到目标位置 ADD 指令不支持身份验证，如果远程文件受身份验证保护，应该使用 RUN wget 或 RUN curl 指令 如果源文件路径，或目标路径包含特殊字符，需要使用 Go 语言的转文规则来转换 如果目标路径以 / 结尾，则默认为目录路径，如果目标目录不存在，ADD 指令会自动创建目录

用法

1

  ADD [--chown=<user>:<group>] <src>... <dest>

或

1

  ADD [--chown=<user>:<group>] ["<src>",... "<dest>"]

参数

参数名 释义
chown 指定文件的用户和用户组
src 源文件，如果是相对路径，则是对应当前构建上下文目录的相对路径源文件路径也可以是远程文件地址源文件路径支持通配符，使用 Go 语言的 filepath.Match 规则来匹配
dest 目标路径，可以是绝对路径，也可以是相对于工作目录的相对路径

示例

1
2

  ADD demo.jar /home
  ADD #.xml config/

COPY

COPY 指令与 ADD 指令相似，但 COPY 指令不会将压缩文件解压缩

用法

1

  COPY [--chown=<user>:<group>] <src>... <dest>

或

1

  COPY [--chown=<user>:<group>] ["<src>",... "<dest>"]

示例

1
2

  COPY demo.jar /home
  COPY #.xml config/

ENTRYPOINT

ENTRYPOINT 指令与 CMD 指令类似，用来指定容器启动时执行的命令，与 CMD 不同的是 ENTRYPOINT 指定的命令不会被 docker run 覆盖

用法

1

  ENTRYPOINT ["executable", "param1", "param2"]

或

1

  ENTRYPOINT command param1 param2

示例

1

  ENTRYPOINT /bin/bash

VOLUME

VOLUME 指令将指定目录挂载为卷，由于 VOLUME 指令不能指定宿主目录，所以挂载卷为宿主 /var/lib/docker/volumes 目录下一个随机名目录 可以通过 docker run -v ~/data:/home/data 来指定宿主挂载目录 挂载卷的好处是容器产生的动态数据可以存储到挂载卷中，这样容器不存储数据，数据都存储在宿主机器上 可以通过 docker inspect 命令查看容器挂载卷

用法

1

  VOLUME ["<path>", "<path>"...]

或

1

  VOLUME <path>

参数

参数名 释义
path 要挂载的容器中的目录

示例

1

  VOLUME ["/home/data"]

WORKDIR

WORKDIR 指令指定容器的工作目录

用法

1

  WORKDIR <path>

示例

1

  WORKDIR /home

ARG

ARG 指令定义一个变量

1

  ARG CENTOS_VERSION=latest

等价于

1

  docker build --build-arg CENTOS_VERSION=latest

用法

1

  ARG <name>[=<default value>]

示例

1
2

  ARG CENTOS_VERSION=latest
  FROM centos:$CENTOS_VERSION

Docker CLI

docker run

从指定镜像运行一个容器，Docker在隔离的容器中运行进程，容器是在主机上运行的进程，主机可以是本地主机，也可以是远程主机 容器进程是独立的，因为它具有自己的文件系统，自己的网络以及独立于主机的独立的进程树

用法

1

  docker run [OPTIONS] IMAGE[:TAG|@DIGEST] [COMMAND] [ARG...]

参数

detach

用法

1

  --detach | -d

在后台运行容器，未指定该参数时默认为前台运行容器，可以使用 docker attach 命令重新附着到容器上 示例

1

  docker run -d -i -t --rm centos

add-host

用法

1

  --add-host <host:ip>

修改容器 /etc/hosts 文件，添加主机到 IP 的映射 示例

1

  docker run -d -i -t --rm --add-host google.com:127.0.0.1 centos

attach

附着到容器的 STDIN、STDOUT、STDERR 等输入输出流，attach 参数不能与 detach 参数同时使用 用法

1

  --attach | -a <stream>

示例

1

  docker run -i -t --rm --attach stdout centos

–cidfile

把容器 ID 打印到指定文件中 用法

1

  --cidfile <cidfile>

示例

1

  docker run -i -t --rm --cidfile cidfile centos

cpus

设置容器 CPU 数量 用法

1

  --cpus <count>

示例

1

  docker run -i -t --rm --cpus 1 centos

dns

修改 /etc/resolv.conf 文件，设置容器 DNS 服务器 用法

1

  --dns <dns>

示例

1

  docker run -i -t --rm --dns 1.1.1.1 --dns 8.8.8.8 centos

entrypoint

覆盖容器默认的 entrypoint 用法

1

  --dns <dns>

示例

1

  docker run -i -t --rm --entrypoint /bin/bash centos

env

设置容器环境变量 用法

1

  --env | -e <key=value>

示例

1

  docker run -i -t --rm --env hello=world centos

hostname

修改 /etc/hostname，设置容器主机名 用法

1

  --hostname  -h <hostname>

示例

1

  docker run -i -t --rm --hostname jason-pc centos

ip

设置容器 IP 地址 用法

1

  --ip <ip>

示例

1

  docker run -i -t --rm --hostname jason-pc centos

label

设置容器元数据 用法

1

  --label | -l <key=value>

示例

1

  docker run -i -t --rm --label hello=world centos

link

链接另一个容器，从而可以与另一个容器通信 link 参数会自动修改 /etc/hosts，添加另一个容器的 host 到 ip 映射 用法

1

  --link <container>

示例

1
2

  docker run -i -t --rm --link nexus centos
  curl http://nexus:8081

memory

设置容器内存大小 用法

1

  --memory | -m <number g | m | k | b>

示例

1

  docker run -i -t --rm --memory 1g centos

name

设置容器名称 用法

1

  --name <name>

示例

1

  docker run -i -t --rm --name centos centos

publish

publish 参数将容器端口绑定到宿主端口，通过宿主端口可以访问到容器端口 用法

1

  --publish | -p <host-port>:<container-port>

示例

1

  docker run -i -t --rm --publish 80:80 centos

restart

设置容器退出时重启策略，默认策略为 no restart 参数不能与 rm 参数同时使用 可先参数值为：

  • no 不重启
  • aways 总是重启 -on-failure 容器以非 0 状态码退出时重启，可以指定尝试重启的最大次数 用法
1

  --restart [no | always | on-failure[:max-retries]]

示例

1
2
3

  docker run -i -t --restart always centos
  exit 0
  docker ps -a

rm

容器退出时自动删除容器 用法

1

  --rm

示例

1

  docker run -i -t --rm centos

tty

开启 TTY 用法

1

  --tty | -t

示例

1

  docker run -i -t --rm centos

volume

挂载卷，将宿主指定目录挂载到容器指定目录，这样对宿主目录下文件的操作就是对容器目录下文件的操作 通过挂载卷，即使容器退出，或删除，容器的文件依然存在宿主机器上 用法

1

  --volume | -v <host-dir>:<container-dir>

示例

1

  docker run -i -t --rm -v ~/Docker/centos/home:/home centos

volume

设置容器工作目录 用法

1

  --workdir | -w <dir>

示例

1

  docker run -i -t --workdir /home centos
