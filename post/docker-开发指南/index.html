Dockerfile

Dockerfile 指令必须大写

FROM

FROM 指令初始化一个新的构建阶段，同时为后续指令指定一个基本镜像，因此，一个合法的 Dockerfile 必须以 FROM 指令开始

用法

FROM <image> [AS <name>]

或

FROM <image>[:<tag>] [AS <name>]

或

FROM <image>[@<digest>] [AS <name>]

参数

参数名 释义
image 镜像名
tag 镜像标签
digest 镜像数字签名，使用 docker images ––digests 查看

示例

ARG CENTOS_VERSION=latest
FROM centos:${CENTOS_VERSION}

RUN

RUN 指令用来指定容器构建时要运行的命令，这些命令是在容器内执行的

用法

RUN <command>

或

RUN ["executable", "param1", "param2"]

示例

RUN ["/bin/bash","-c","echo runing > log.txt"]

RUN echo runing >> log-1.txt

CMD

CMD 指令用来指定容器启动时的命令，这和 docker run 命令非常类似，如果 docker run 指定了执行命令，将会覆盖 CMD 指令的命令。

这些命令是在容器内执行的，一个 Dockerfile 只能有一个 CMD 指令，如果有多个，则只执行最后一个

CMD /bin/bash

等价于

docker run -t -i centos /bin/bash

用法

CMD ["executable","param1","param2"]

或

CMD ["param1","param2"]

或

CMD command param1 param2

示例

CMD /bin/bash

LABEL

LABEL 指令为镜像添加元数据，可通过 docker inspect 命令查看

用法

LABEL <key>=<value> <key>=<value> <key>=<value> ...

示例

LABEL label1=label1 label2="this is lable2"

EXPOSE

EXPOSE 指令用于声明容器将暴露的端口及协议类型，实际上并未发布端口，它充当构建镜像的人员与运行容器的人员之间的一种文档类型，运行容器时使用 docker run -p 来映射容器端口和宿主端口的绑定

可以使用 docker inspect 命令查看镜像暴露的端口

用法

EXPOSE <port> [<port>/<protocol>...]

参数

参数名 释义
port 端口
protocol 通信协议，TCP 或 UDT，默认TCP

示例

EXPOSE 8080

ENV

ENV 指令用于设置环境变量，可以使用 docker inspect 命令查看镜像设置的环境变量

ENV key1=value1 key2=value2

等价于

docker run --env key1=value1 --env key2=value2

用法

ENV <key> <value>

或

ENV <key>=<value> ...

示例

ENV hello world

ADD

ADD 指令复制宿主上的文件，或远程文件到容器中，如果源文件是压缩文件（identity, gzip, bzip2 或 xz 格式），ADD 指令会将源文件解压到目标位置

ADD 指令不支持身份验证，如果远程文件受身份验证保护，应该使用 RUN wget 或 RUN curl 指令

如果源文件路径，或目标路径包含特殊字符，需要使用 Go 语言的转文规则来转换

如果目标路径以 / 结尾，则默认为目录路径，如果目标目录不存在，ADD 指令会自动创建目录

用法

ADD [--chown=<user>:<group>] <src>... <dest>

或

ADD [--chown=<user>:<group>] ["<src>",... "<dest>"]

参数

参数名 释义
chown 指定文件的用户和用户组
src 源文件，如果是相对路径，则是对应当前构建上下文目录的相对路径
源文件路径也可以是远程文件地址
源文件路径支持通配符，使用 Go 语言的 filepath.Match 规则来匹配
dest 目标路径，可以是绝对路径，也可以是相对于工作目录的相对路径

示例

ADD demo.jar /home
ADD *.xml config/

COPY

COPY 指令与 ADD 指令相似，但 COPY 指令不会将压缩文件解压缩

用法

COPY [--chown=<user>:<group>] <src>... <dest>

或

COPY [--chown=<user>:<group>] ["<src>",... "<dest>"]

示例

COPY demo.jar /home
COPY *.xml config/

ENTRYPOINT

ENTRYPOINT 指令与 CMD 指令类似，用来指定容器启动时执行的命令，与 CMD 不同的是 ENTRYPOINT 指定的命令不会被 docker run 覆盖

用法

ENTRYPOINT ["executable", "param1", "param2"]

或

ENTRYPOINT command param1 param2

示例

ENTRYPOINT /bin/bash

VOLUME

VOLUME 指令将指定目录挂载为卷，由于 VOLUME 指令不能指定宿主目录，所以挂载卷为宿主 /var/lib/docker/volumes 目录下一个随机名目录

可以通过 docker run -v ~/data:/home/data 来指定宿主挂载目录

挂载卷的好处是容器产生的动态数据可以存储到挂载卷中，这样容器不存储数据，数据都存储在宿主机器上

可以通过 docker inspect 命令查看容器挂载卷

用法

VOLUME ["<path>", "<path>"...]

或

VOLUME <path>

参数

参数名 释义
path 要挂载的容器中的目录

示例

VOLUME ["/home/data"]

WORKDIR

WORKDIR 指令指定容器的工作目录

用法

WORKDIR <path>

示例

WORKDIR /home

ARG

ARG 指令定义一个变量

ARG CENTOS_VERSION=latest

等价于

docker build --build-arg CENTOS_VERSION=latest

用法

ARG <name>[=<default value>]

示例

ARG CENTOS_VERSION=latest
FROM centos:$CENTOS_VERSION

Docker CLI

docker run

从指定镜像运行一个容器，Docker在隔离的容器中运行进程，容器是在主机上运行的进程，主机可以是本地主机，也可以是远程主机

容器进程是独立的，因为它具有自己的文件系统，自己的网络以及独立于主机的独立的进程树

用法

docker run [OPTIONS] IMAGE[:TAG|@DIGEST] [COMMAND] [ARG...]

参数

  • detach

    用法

    --detach | -d

    在后台运行容器，未指定该参数时默认为前台运行容器，可以使用 docker attach 命令重新附着到容器上

    示例

    docker run -d -i -t --rm centos

  • add-host

    用法

    --add-host <host:ip>

    修改容器 /etc/hosts 文件，添加主机到 IP 的映射

    示例

    docker run -d -i -t --rm --add-host google.com:127.0.0.1 centos

  • attach

    附着到容器的 STDIN、STDOUT、STDERR 等输入输出流，attach 参数不能与 detach 参数同时使用

    用法

    --attach | -a <stream>

    示例

    docker run -i -t --rm --attach stdout centos

  • –cidfile

    把容器 ID 打印到指定文件中

    用法

    --cidfile <cidfile>

    示例

    docker run -i -t --rm --cidfile cidfile centos

  • cpus

    设置容器 CPU 数量

    用法

    --cpus <count>

    示例

    docker run -i -t --rm --cpus 1 centos

  • dns

    修改 /etc/resolv.conf 文件，设置容器 DNS 服务器

    用法

    --dns <dns>

    示例

    docker run -i -t --rm --dns 1.1.1.1 --dns 8.8.8.8 centos

  • entrypoint

    覆盖容器默认的 entrypoint

    用法

    --dns <dns>

    示例

    docker run -i -t --rm --entrypoint /bin/bash centos

  • env

    设置容器环境变量

    用法

    --env | -e <key=value>

    示例

    docker run -i -t --rm --env hello=world centos

  • hostname

    修改 /etc/hostname，设置容器主机名

    用法

    --hostname  -h <hostname>

    示例

    docker run -i -t --rm --hostname jason-pc centos

  • ip

    设置容器 IP 地址

    用法

    --ip <ip>

    示例

    docker run -i -t --rm --hostname jason-pc centos

  • label

    设置容器元数据

    用法

    --label | -l <key=value>

    示例

    docker run -i -t --rm --label hello=world centos

  • link

    链接另一个容器，从而可以与另一个容器通信

    link 参数会自动修改 /etc/hosts，添加另一个容器的 host 到 ip 映射

    用法

    --link <container>

    示例

    docker run -i -t --rm --link nexus centos

curl http://nexus:8081

  • memory

    设置容器内存大小

    用法

    --memory | -m <number g | m | k | b>

    示例

    docker run -i -t --rm --memory 1g centos

  • name

    设置容器名称

    用法

    --name <name>

    示例

    docker run -i -t --rm --name centos centos

  • publish

    publish 参数将容器端口绑定到宿主端口，通过宿主端口可以访问到容器端口

    用法

    --publish | -p <host-port>:<container-port>

    示例

    docker run -i -t --rm --publish 80:80 centos

  • restart

    设置容器退出时重启策略，默认策略为 no

    restart 参数不能与 rm 参数同时使用

    可先参数值为：

    • no 不重启
    • aways 总是重启

    -on-failure 容器以非 0 状态码退出时重启，可以指定尝试重启的最大次数

    用法

    --restart [no | always | on-failure[:max-retries]]

    示例

    docker run -i -t --restart always centos
exit 0
docker ps -a

  • rm

    容器退出时自动删除容器

    用法

    --rm

    示例

    docker run -i -t --rm centos

  • tty

    开启 TTY

    用法

    --tty | -t

    示例

    docker run -i -t --rm centos

  • volume

    挂载卷，将宿主指定目录挂载到容器指定目录，这样对宿主目录下文件的操作就是对容器目录下文件的操作

    通过挂载卷，即使容器退出，或删除，容器的文件依然存在宿主机器上

    用法

    --volume | -v <host-dir>:<container-dir>

    示例

    docker run -i -t --rm -v ~/Docker/centos/home:/home centos

  • volume

    设置容器工作目录

    用法

    --workdir | -w <dir>

    示例

    docker run -i -t --workdir /home centos

docker attach

将宿主输入、输出、错误流附着到正在运行的容器上

用法

docker attach [OPTIONS] CONTAINER

参数

  • no-stdin

    不附着输入流

    用法

    --no-stdin

    示例

    docker run -i -t --rm -d --name centos centos

docker attach centos

docker build

从 Dockerfile 构建镜像

用法

docker build [OPTIONS] PATH | URL | -

参数

  • file

    指定 Dockerfile 的名称

    用法

    --file | -f <file>

    示例

    docker build -t jason/demo:latest --file Dfile .

  • iidfile

    打印镜像 ID 到指定文件

    用法

    --iidfile

    示例

    docker build -t jason/demo:latest --iidfile iid .

  • tag

    设置镜像名和标签

    用法

    --tag | -t <image-name>[:tag]

    示例

    docker build -t jason/demo:latest .

docker commit

根据窗口的更改创建新的镜像

用法

docker commit [OPTIONS] CONTAINER [REPOSITORY[:TAG]]

参数

  • author

    设置镜像作者

    用法

    --author | -a <author>

    示例

    docker run -i -t --name centos centos
touch hello.txt
exit
docker commit --author jason --message "hello world" --pause centos centos:hello
docker run -i -t --name hello centos:hello

  • message

    设置提交信息，可通过 docker history 命令查看

    用法

    --message | -m <message>
    • pause

    提交镜像期间暂停容器

    用法

    --pause | -p

docker cp

在容器和宿主文件系统之间复制文件或文件夹

用法

docker cp [OPTIONS] CONTAINER:SRC_PATH DEST_PATH | -

或

docker cp [OPTIONS] SRC_PATH | - CONTAINER:DEST_PATH

示例

docker cp ~/test.txt centos:/home
docker cp centos:/home/test.txt ~/

docker create

从镜像创建一个容器，参数与 docker run 命令相同，但只创建不运行

用法

docker create [OPTIONS] IMAGE [COMMAND] [ARG...]

示例

docker create centos --name centos

docker diff

检查容器文件系统的变化

用法

docker diff CONTAINER

示例

docker diff centos

docker exec

在运行的容器中执行命令

用法

docker exec [OPTIONS] CONTAINER COMMAND [ARG...]

参数

  • detach

    分离模式，后台运行命令，这样控制台不会输出任何信息

    用法

    --detach | -d

    示例

    docker exec --detach centos free

  • interactive

    保持 STDIN 标准输入流打开

    用法

    --interactive | -i

    示例

    docker exec -i centos free

  • tty

    开启 TTY

    用法

    --tty | -t

    示例

    docker exec -i -t centos free

  • workdir

    设置工作目录

    用法

    --workdir | -w <path>

    示例

    docker exec --workdir /home centos ls

docker export

导出容器文件系统到 tar 压缩文件

用法

docker export [OPTIONS] CONTAINER

参数

  • output

    指定输出文件

    用法

    --output | -o <tar-file>

    示例

    docker export --output centos.tar centos

docker images

列出镜像

用法

docker images [OPTIONS] [REPOSITORY[:TAG]]

参数

  • all

    显示所有镜像

    用法

    --all | -a

    示例

    docker images -a

  • digests

    显示数字签名

    用法

    --digests

    示例

    docker images --digests

docker info

显示 docker 系统信息

用法

docker info [OPTIONS]

docker inspect

显示镜像或容器信息

用法

docker inspect [OPTIONS] NAME | ID [NAME | ID...]

示例

docker inspect centos

docker kill

杀死一个或多个容器

用法

docker kill [OPTIONS] CONTAINER [CONTAINER...]

参数

  • signal

    发送给容器的信息

    用法

    --signal | -s

    示例

    docker kill -s 9 centos

docker logs

显示容器日志

用法

docker logs [OPTIONS] CONTAINER

参数

  • details

    显示额外细节信息

    用法

    --details

    示例

    docker logs --details -f centos

  • follow

    跟随日志输出

    用法

    --follow | -f

    示例

    docker logs --details -f centos

  • since

    指定日志开始时间

    用法

    --since

    示例

    docker logs -f --since 2019-11-26T15:40:00 centos

  • tail

    显示末尾 N 行日志

    用法

    --tail <number>

    示例

    docker logs -f --tail 1 centos

  • until

    显示指定时间之前的日志

    用法

    --until

    示例

    docker logs --until 2019-11-26T15:53:00 centos

docker pause

暂停一个或多个容器

用法

docker pause CONTAINER [CONTAINER...]

示例

docker pause centos1 centos2

docker unpause

恢复一个或多个已暂停的容器

用法

docker unpause CONTAINER [CONTAINER...]

示例

docker unpause centos1 centos2

docker port

列出容器端口与宿主端口的映射，或者指定一个容器端口，显示这个容器端口与宿主端口的映射

用法

docker port CONTAINER [PRIVATE_PORT[/PROTO]]

示例

docker port centos

docker port centos 8080/tcp

docker ps

列出容器

用法

docker ps [OPTIONS]

参数

  • all

    列出所有容器

    用法

    --all | -a

    示例

    docker ps -a

  • size

    显示文件大小

    用法

    --size | -s

    示例

    docker ps -s

docker pull

从 docker 远程仓库下载一个镜像到本地

用法

docker pull [OPTIONS] NAME[:TAG | @DIGEST]

参数

  • all-tags

    下载所有标签的镜像到本地

    用法

    --all-tags | -a

    示例

    docker pull --all-tags centos

  • disable-content-trust

    跳过镜像检查

    用法

    --disable-content-trust

    示例

    docker pull --disable-content-trust centos

  • quiet

    静默下载，不输出日志

    用法

    --quiet | -q

    示例

    docker pull --quiet centos

docker rename

重命名容器

用法

docker rename CONTAINER NEW_NAME

示例

docker rename centos centos-renamed

docker restart

重启一个或多个容器

用法

docker restart [OPTIONS] CONTAINER [CONTAINER...]

示例

docker restart centos

docker rm

删除一个或多个容器

用法

docker rm [OPTIONS] CONTAINER [CONTAINER...]

参数

  • force

    强制删除容器，即使容器正在运行

    用法

    --force | -f

    示例

    docker rm -f centos

docker rmi

删除一个或多个镜像

用法

docker rmi [OPTIONS] IMAGE [IMAGE...]

参数

  • force

    强制删除镜像，即使这个镜像有正在运行的容器

    用法

    --force | -f

    示例

    docker rmi -f centos

docker save

把一个或多个镜像导出为 tar 压缩文件（默认导出到 STDOUT）

用法

docker save [OPTIONS] IMAGE [IMAGE...]

参数

  • output

    指定输出文件

    用法

    --output | -o <file>

    示例

    docker save -o ~/centos.tar centos

docker import

从 tar 压缩文件导入镜像

用法

docker import [OPTIONS] file|URL|- [REPOSITORY[:TAG]]

示例

docker import ~/centos.tar centos:latest

docker search

从 Docker Hub 搜索镜像

用法

docker search [OPTIONS] TERM

参数

  • filter

    设置过滤条件

    用法

    --filter | -f <key=value>'

    示例

    #搜索标星大于10、非自动构建、官方的 centos 镜像
docker search --filter "stars=10" --filter "is-automated=false" --filter "is-official=true" centos

  • limit

    限制搜索结果数量

    用法

    --limit

    示例

    docker search --limit 5 centos

docker start

启动一个已停止的容器

用法

docker start [OPTIONS] CONTAINER [CONTAINER...]

参数

  • attach

    附着到容器 STDOUT、STDERR

    用法

    --attach | -a

    示例

    docker start -a centos

  • interactive

    附着到容器 STDIN

    用法

    --interactive | -i

    示例

    docker start -a -i centos

docker stats

显示容器资源使用量的统计

用法

docker stats [OPTIONS] [CONTAINER...]

参数

  • all

    显示所有容器

    用法

    --all | -a

    示例

    docker stats -a

docker stats centos

docker stop

停止一个正在运行的容器

用法

docker stop [OPTIONS] CONTAINER [CONTAINER...]

示例

docker stop centos

docker tag

为一个镜像打标签

用法

docker tag SOURCE_IMAGE[:TAG] TARGET_IMAGE[:TAG]

示例

docker tag centos:latest centos:demo

docker top

显示容器中正在运行的进程

用法

docker top CONTAINER [ps OPTIONS]

示例

docker top centos
