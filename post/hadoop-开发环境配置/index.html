注意：两台主机的用户名必须一样

slave1jason10.2.2.178datanode

软件概览

软件版本

环境变量配置（两台机器都要配置）

编辑/etc/profile文件

#JAVA环境变量begin
export JAVA_HOME=/opt/jdk1.8.0_65
export JRE_HOME=${JAVA_HOME}/jre
export CLASSPATH=.:${JAVA_HOME}/lib:${JRE_HOME}/lib
export PATH=${JAVA_HOME}/bin:$PATH
#JAVA环境变量end

#hadoop环境变量begin
export HADOOP_HOME=/opt/hadoop
export HADOOP_CONF_DIR=${HADOOP_HOME}/etc/hadoop
export PATH=$PATH:${HADOOP_HOME}/bin
export PATH=$PATH:${HADOOP_HOME}/sbin
#hadoop环境变量end

配置免密码登陆

  1. 生成密钥

    cd ~/.ssh
ssh-keygen -t dsa

这里生成私钥id_dsa和公钥id_dsa.pub

  1. master主机设置免密码登陆

    cat id_dsa.pub >> ~/.ssh/authorized_keys

  2. slave1主机设置免密码登陆

    scp id_dsa.pub jason@slave1:~/.ssh
ssh jason@slave1
cd .ssh
cat id_dsa.pub >> ~/.ssh/authorized_keys

配置Hadoop配置文件（两台机器都要配置）

  1. 编辑mapred-site.xml

    <configuration>
  <property>
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
  </property>
</configuration>

  2. 编辑core-site.xml

    <configuration>
  <property>
    <name>fs.defaultFS</name>
    <value>hdfs://master:9000</value>
  </property>
  <property>
    <name>hadoop.tmp.dir</name>
    <value>/home/jason/other/hadoop/tmp</value>
  </property>
</configuration>

  3. 编辑slaves

    slave1

  4. 编辑hadoop-env.sh

    export JAVA_HOME=/opt/jdk1.8.0_65

  5. 编辑hdfs-site.xml

    <configuration>
  <property>
    <name>dfs.namenode.secondary.http-address</name>
    <value>Master:9001</value>
  </property>
  <property>
    <name>dfs.replication</name>
    <value>1</value>
  </property>
  <property>
    <name>dfs.namenode.name.dir</name>
    <value>file:/home/jason/other/hadoop/name</value>
  </property>
  <property>
    <name>dfs.datanode.data.dir</name>
    <value>file:/home/jason/other/hadoop/data</value>
  </property>
</configuration>

  6. 编辑yarn-site.xml

    <configuration>
  <property>
    <name>yarn.resourcemanager.hostname</name>
    <value>master</value>
  </property>
  <property>
    <name>yarn.nodemanager.aux-services</name>
    <value>mapreduce_shuffle</value>
  </property>
</configuration>

格式化namenode

hdfs namenode -format

start-all.sh
